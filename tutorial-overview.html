Overview

Now that you’re set up locally, it’s time to take a peek at React. Don’t worry, React doesn’t bite. (The error messages sometimes do.)

Three things you want to keep open

  1. VS Code — where you edit your files (your glorious code cavern).
  2. Your terminal — where you run the dev server (usually npm run dev for Vite).
  3. Your browser — where you see your app at http://localhost:5173 (or whatever it prints).

When you save a file in VS Code, the browser usually updates automatically. To make it even easier you can install Live Preview from Microsoft which is what I use

The starter code

In your project, open src/App.jsx (or src/App.js depending on your setup). It should start with something small and suspiciously calm, like: 

export default function Square() {
  return <button className="square">X</button>;
}

If your browser shows a single square with an X, congrats: you have successfully summoned React.

What even is a component?

This is a component: 

export default function Square() {
  return <button className="square">X</button>;
}

A component is just a reusable chunk of UI. Think “custom HTML tag,” except it can have logic and state and ambitions.

  • function Square() defines the component.
  • export default means other files can import it, and it’s the “main export” of this file.
  • The return is what shows up on the page.
  • <button> here is written in JSX (HTML-ish syntax inside JavaScript).
  • className="square" connects it to CSS (React uses className, not class).

styles.css (aka: “make it not ugly”)

Your styles live in something like src/styles.css. The .square class is what makes your button actually look like a tic-tac-toe square.

main.jsx / index.js (the “plugs React into the page” file)

You probably also have src/main.jsx (Vite) or src/index.js (other setups). You won’t edit it much, but it’s the bridge between your components and the browser.

Building the board

If you try to return two buttons side-by-side like this: 

export default function Square() {
  return <button className="square">X</button><button className="square">X</button>;
}

React will complain because a component must return one parent element. The fix is a Fragment: <> ... </> 

export default function Square() {
  return (
    <>
      <button className="square">X</button>
      <button className="square">X</button>
    </>
  );
}

Next you’ll make 9 squares, realize they’re in one long line (pain), and then group them into rows: 

export default function Board() {
  return (
    <>
      <div className="board-row">
        <button className="square">1</button>
        <button className="square">2</button>
        <button className="square">3</button>
      </div>
      <div className="board-row">
        <button className="square">4</button>
        <button className="square">5</button>
        <button className="square">6</button>
      </div>
      <div className="board-row">
        <button className="square">7</button>
        <button className="square">8</button>
        <button className="square">9</button>
      </div>
    </>
  );

}

Passing data with props

Instead of writing 9 different buttons by hand forever, we make a reusable Square component, and pass it different values using props. 

function Square({ value }) {
  return <button className="square">{value}</button>;
}

export default function Board() {
  return (
    <>
      <div className="board-row">
        <Square value="1" />
        <Square value="2" />
        <Square value="3" />
      </div>
      <div className="board-row">
        <Square value="4" />
        <Square value="5" />
        <Square value="6" />
      </div>
      <div className="board-row">
        <Square value="7" />
        <Square value="8" />
        <Square value="9" />
      </div>
    </>
  );
}

The curly braces {value} are the “escape hatch” from JSX into JavaScript. Without them, React thinks you literally want the word value.

Making it interactive (it clicks!)

Next you’ll add an onClick handler: 

function Square({ value }) {
  function handleClick() {
    console.log('clicked!');
  }

  return (
    <button className="square" onClick={handleClick}>
      {value}
    </button>
  );
}

If you’re running locally, open your browser DevTools Console to see that log.

State (a square learns to remember)

Logging clicks is cute, but we want the square to remember it was clicked. That’s state, and React gives us useState. 

import { useState } from 'react';

function Square() {
  const [value, setValue] = useState(null);

  function handleClick() {
    setValue('X');
  }

  return (
    <button className="square" onClick={handleClick}>
      {value}
    </button>
  );
}

Every time you call setValue, React re-renders the component. That’s the whole trick. That’s the secret sauce. That’s the entire spellbook.

React DevTools (optional but elite)

If you want superpowers, install the React Developer Tools browser extension. Then you can inspect your components and their props/state directly.
