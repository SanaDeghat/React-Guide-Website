Okay… but why not just write normal HTML/JS?

1) Because building websites gets complicated fast

A simple page? Static wesbsites? Thats light work But the moment your website needs to update itself, then things can get really messy really fast

2) The “manual DOM” problem

In plain Java Script youre forced to do a lot of:

“find the element → change the text → hide this → show that → remember the state yourself”.

this usually works untill it doenst. and when it doesny its really hard to debug

3) React’s strength: building UI from components

Components are meant to be like like LEGO bricks for your interface: small pieces you can reuse, nest, and move around without rewriting the whole website like a tragic novel.

4) React updates the screen for you

you describe what the UI should look like for a given state and react makes it happen when the state comes.

5) Bonus: Scalibility

Its really easy to scale, What else could you ask for?
