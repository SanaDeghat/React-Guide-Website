Okay… but why not just write normal HTML/JS?

1) Because building websites gets complicated fast

A simple page? Static websites? That's light work. But the moment your website needs to update itself, things can get really messy, really fast.

2) The “manual DOM” problem

In plain JavaScript you're forced to do a lot of:

“find the element → change the text → hide this → show that → remember the state yourself”.

This usually works until it doesn't. And when it doesn't, it's really hard to debug.

3) React’s strength: building UI from components

Components are meant to be like LEGO bricks for your interface: small pieces you can reuse, nest, and move around without rewriting the whole website like a tragic novel.

4) React updates the screen for you

You describe what the UI should look like for a given state and React makes it happen when the state changes.

5) Bonus: Scalability

It's really easy to scale. What else could you ask for?
