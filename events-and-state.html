Respond to events and update the screen

Responding to events

You can respond to events by declaring event handler functions inside your components. Here’s a click handler:

function MyButton() { function handleClick() { alert('You clicked me!'); } return ( <button onClick={handleClick}> Click me </button> ); }

Notice how onClick={handleClick} has no parentheses. Don’t call the handler — just pass it. React calls it when the user clicks.

Updating the screen with state

If you want your component to “remember” something (like a click count), you use state.

First, import useState from React:

import { useState } from 'react';

Then declare state inside the component:

function MyButton() { const [count, setCount] = useState(0); // ... }

Example: increment a counter on click:

function MyButton() { const [count, setCount] = useState(0); function handleClick() { setCount(count + 1); } return ( <button onClick={handleClick}> Clicked {count} times </button> ); }

Each component instance gets its own state

If you render the same component multiple times, each one has its own independent state:

import { useState } from 'react'; export default function MyApp() { return ( <div> <h1>Counters that update separately</h1> <MyButton /> <MyButton /> </div> ); } function MyButton() { const [count, setCount] = useState(0); function handleClick() { setCount(count + 1); } return ( <button onClick={handleClick}> Clicked {count} times </button> ); }

Hooks rules (important)

Functions starting with use are called Hooks. Hooks are more restrictive than regular functions:

  • Only call Hooks at the top level of your component (not inside loops, conditions, or nested functions)
  • If you need a hook inside a conditional UI path, extract a new component
